# 笔记风格

  • 用空格比逗号好
  • 用换行比句号好
  • 不用[xxx](http://)隐藏链接(除非太长) --> 网址本身也是一种信息
  • 不分h1-h6 --> @lazy
  • 用分隔线适时地划分逻辑内容
  • 每篇文章最好有张图片

# 目录层级

  • 最多9个chapter
  • 最多5个topic
  • 最多1个appendix(主体内容不能放到这里)
[0]collection 
    ch9-chapter 
        topic5 
            _appendix1 

// appendix如下: 
private --> 私人内容, 不方便公开的
cmpt, case, demo, tricks, steps, tag --> 组件/案例/示例/技巧总结/步骤/标签
record, idea, faq, todo --> 按照时间轴记录
notnow, old, offcut, doubt --> 不推荐的/过时内容
snippets, worm --> 代码片段/文章选段
spec --> 标准规范
corner, minor --> 少数派
